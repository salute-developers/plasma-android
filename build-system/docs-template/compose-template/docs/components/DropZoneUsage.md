DropZone - Интерактивная область, предназначенная для перетаскивания и загрузки файлов.

Представляет собой контейнер, в котором, в качестве подсказок, возможно разместить заголовок, описание 
и контент (контент может быть любым @composable). Если в качестве контента провайдится иконка, она будет  
автоматически стилизована.

## Состояние DropZone

Для управления состоянием используется DropZoneState. Это семантические состояния для изменения внешнего вида  
компонента в процессе взаимодействия с ним. Компонент стилизуется автоматически при передаче актуального состояния

## Расположение контента (иконки) в DropZone

Расположение контента задается при помощи DropZoneIconPlacement, возможны три сценария  
- контент расположен над заголовком
- контент расположен вначале, на одном уровне с заголовком
- контент не отображается

## Обводка DropZone

Для визуального эффекта в DropZone используется обводка по границе компонента. При взаимодействии и  
передачи в DropZone актуального DropZoneState, обводка стилизуется автоматически. Изменение внешнего вида  
бордера возможно при мощи DropZoneBorderType
- в виде прерывистой линии (пунктирной)
- в виде сплошной линии
- не отображать

## OnClick() 

В DropZone предусмотрен параметр `onClick: (() -> Unit)? = null`, с помощью которого, на стороне пользователя  
возможна настройка поведения при клике по компоненту. Например это может быть полезным в сценарии, при котором  
кликая по области DropZone открывается файловый менеджер для выбора файла.

## Стиль DropZone

В большинстве случаев можно использовать готовые сгенерированные стили, а при необходимости создать
собственный стиль через соответствующий builder. Подробнее о том, как работают `Style`, `StyleBuilder`
и stateful-параметры стиля, см. в разделе [Стилизация компонентов](../theme/Styles.md).
