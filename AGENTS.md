# AGENTS.md

Этот документ описывает основные компоненты и агенты проекта `plasma-android`, которые обеспечивают реализацию и поддержку дизайн-системы SDDS на платформе Android.

## Обзор

Проект `plasma-android` объединяет набор решений для адаптации дизайн-системы SDDS в Android-приложениях. Он включает в себя библиотеки, плагины и инструменты, обеспечивающие единый визуальный стиль, компоненты интерфейса и процессы сборки.

## Основные компоненты

### 1. sdds-core

Ядро любой дизайн-системы. Расположенно в директории `sdds-core/`. Включает в себя:
- **Theme Builder Plugin**: Плагин для генерации токенов и компонентов дизайн-систем .
- **SDDS UIKit**: Базовые UI-компоненты для Android XML.
- **SDDS Compose UIKit**: Базовые UI-компоненты для Jetpack Compose.

### 2. integration-core

Инфраструктурные библиотеки для создания дизайн-систем, расположенное в директории `integration-core/`. Включает в себя:
- **sandbox-core**: Общие сущности для разработки демо-приложений.
- **sandbox-ksp**: KSP-плагин генерации boilerplate-кода для разработки демо-приложений.
- **sandbox-compose**: Содержит UI-каркас для разработки демо-приложений для Jetpack Compose.
- **sandbox-view**: Содержит UI-каркас для разработки демо-приложений для Android XML.
- **uikit-compose-fixtures**: Общие фикстуры для документации и демо-приложений на Jetpack Compose - примеры использования компонентов, экраны (Story) для демо-приложений.
- **uikit-compose-testcases**: Общие фикстуры для тестирования на Jetpack Compose - тест кейсы для скриншотного тестирования и обвязка для скриншотов документации.
- **uikit-fixtures**: Общие фикстуры для документации и демо-приложений на Android View - примеры использования компонентов, экраны (Story) для демо-приложений.
- **uikit-testcases**: Общие фикстуры для тестирования на Android View - тест кейсы для скриншотного тестирования и обвязка для скриншотов документации.

### 3. build-system

Набор Gradle-плагинов и конфигураций, обеспечивающих единые процессы сборки, тестирования и публикации модулей.

### 4. tokens

Содержит различные дизайн-системы. Расположен в директориях `tokens/*`.
Каждый модуль - это самостоятельная дизайн-система. Суффикс вконце названия модуля указывает на целевую платформу (Compose / View).
В каждом модуле есть подмодули следующие подмодули: 
- **app** - демо-приложение для этой системы.
- **integration** - интеграционный модуль для этой системы. Здесь перечислены вариации компонентов и настройка темы для демо-приложения.
- **docs** - документация для этой системы, с примерами кода.

90% всего кода здесь генерируется при помощи Plugin Theme Builder.

## Зависимости

Проект использует версионирование зависимостей через `libs.versions.toml`. Ключевые версии:
- **Kotlin**: 1.9.25
- **AndroidX Compose BOM**: 2025.05.00
- **SDDS UIKit**: 0.40.0
- **SDDS Compose**: 0.42.0

## Инструменты

- **Статический анализ**: detekt, ktlint, Spotless
- **Тестирование**: JUnit, MockK, Robolectric, Espresso, Paparazzi, Roborazzi
- **Плагины**: KSP, Dokka, Metalava, Maven Publish

## Структура репозитория

- `sdds-core/` — основные библиотеки и плагины дизайн-системы.
- `integration-core/` — основные библиотеки для создания инфраструктуры дизайн-систем (демо-приложений, документации).
- `build-system/` — общие конфигурации сборки.
- `gradle/libs.versions.toml` — централизованное управление версиями.
- `playground/` - проект для экспериментов с новыми фичами.
- `tokens/` — библиотеки токенов и компонентов клиентских дизайн-систем.

Документация по отдельным компонентам доступна в соответствующих README.md файлах.
