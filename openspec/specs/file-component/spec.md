## ADDED Requirements

### Requirement: File рисует фон с учётом формы
Composable `File` SHALL отрисовывать фон корневого контейнера кистью `style.colors.backgroundColor`, разрешённой по `interactionSource`, обрезая его по форме `style.shape`, разрешённой по `interactionSource`.

#### Scenario: Стиль задаёт непрозрачный фон и форму
- **WHEN** в стиле заданы `backgroundColor` (непрозрачный) и `shape` (например, скруглённая форма)
- **THEN** под контентом File отрисовывается заливка указанной кистью, ограниченная указанной формой

#### Scenario: Фон/форма меняются по состоянию
- **WHEN** для `backgroundColor`/`shape` заданы разные значения для разных состояний и меняется состояние `interactionSource`
- **THEN** File перерисовывает фон/форму в соответствии с текущим состоянием

#### Scenario: Дефолтный стиль не рисует видимого фона
- **WHEN** стиль использует дефолты (прозрачная кисть, `RectangleShape`)
- **THEN** внешний вид File не отличается от поведения до изменения — видимой заливки и скругления нет

### Requirement: File применяет внешние отступы вокруг контента
Composable `File` SHALL применять к корневому контейнеру внешние отступы из `style.dimensions.getPaddingValues(interactionSource)` (раздельные `paddingStart`/`paddingTop`/`paddingEnd`/`paddingBottom`). Отступы применяются поверх фона, то есть фон занимает всю область компонента, а контент смещается внутрь на величину отступов.

#### Scenario: Стиль задаёт ненулевые отступы
- **WHEN** в стиле заданы ненулевые отступы для сторон
- **THEN** контент File смещается внутрь на заданные значения, а фон по-прежнему покрывает всю область компонента включая отступы

#### Scenario: Дефолтные отступы равны нулю
- **WHEN** стиль использует дефолтные отступы (`0.dp` для всех сторон)
- **THEN** раскладка File совпадает с поведением до изменения
